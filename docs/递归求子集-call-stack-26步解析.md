# 递归求子集：用 26 步看懂 call stack 的完整变化

这篇文档解析的是 LeetCode 78「子集」问题。输入 `nums = [1, 2, 3]`，目标是输出它的所有子集：

```text
[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]
```

代码的核心函数是：

```cpp
void subsets(vector<int>& nums, vector<int>& sub, vector<vector<int>>& ans, int start) {
    ans.emplace_back(sub);

    for (int i = start; i < nums.size(); i++) {
        sub.emplace_back(nums[i]);
        subsets(nums, sub, ans, i + 1);
        sub.pop_back();
    }
}
```

这段递归代码最容易卡住的地方，不是语法，而是脑子里没有一张清晰的 call stack 图。我们可以把它理解成一句话：

**每进入一次 `subsets`，就先把当前路径 `sub` 收进答案；然后从 `start` 往后试每一个数，选它、递归、回来后撤销选择。**

## 先理解 3 个变量

`nums` 是原始数组，固定为 `[1, 2, 3]`。

`sub` 是当前正在走的路径，也就是“我现在已经选了哪些数”。它会随着递归不断 `push` 和 `pop`。

`start` 表示下一次只能从哪个下标开始选。这个变量很关键：它保证不会回头选已经考虑过的数，所以不会出现 `[2,1]` 这种重复排列。

`ans` 是最终答案。注意代码一进入函数就执行 `ans.emplace_back(sub)`，所以每个递归节点都会贡献一个子集。

## 为什么先把 `sub` 放进 `ans`

求子集不是求“长度固定”的组合。任何长度都可以，包括空集、一个元素、两个元素、三个元素。

所以，只要递归走到一个节点，当前 `sub` 就是一个合法子集：

```text
进入 root: sub = []       -> 收集 []
进入 1 分支: sub = [1]    -> 收集 [1]
进入 1,2 分支: sub=[1,2] -> 收集 [1,2]
```

后面还能不能继续选，是另一件事；当前路径本身已经是一个答案。

## 26 步完整过程

飞书画板里有 9 个主要表格：`main()`，以及 8 次 `subsets()` 调用。每个 `subsets()` 表格都代表一个栈帧，也就是一次函数调用现场。

画板里的箭头编号 1 到 26，可以按下面的方式读：

- 从一个表格指向另一个表格：表示函数调用进入新栈帧，或子函数返回到上一个栈帧。
- 指向同一个表格的回环箭头：表示仍在同一个栈帧里继续执行，比如递归返回后执行 `pop_back()`，或者 `for` 循环进入下一轮。
- 每个 `subsets()` 表格顶部的 `sub`、`ans`、`start` 是刚进入这个栈帧时的状态；表格中间/下方的 `sub{...}` 和 `ans{...}` 是执行到对应位置后的状态。

下面按照画板箭头编号，把这 26 步完整走一遍。

### 第 1 组：从 `main()` 进入根栈帧，再沿着 1 -> 2 -> 3 走到底

| 画板步骤 | 从哪里到哪里 | 发生了什么 | 关键状态 |
| --- | --- | --- | --- |
| 1 | `main()` -> `subsets(sub{})` | `main()` 调用 `subsets(nums, sub, ans, 0)`，创建根栈帧 | `sub = {}`，`start = 0`，`ans` 还空 |
| 2 | `subsets(sub{})` -> `subsets(sub{1})` | 根栈帧先收集空集，然后 `i=0`，执行 `push 1`，递归进入下一层 | 新栈帧 `sub = {1}`，`start = 1` |
| 3 | `subsets(sub{1})` -> `subsets(sub{1,2})` | `{1}` 栈帧先把 `{1}` 加入答案，然后 `i=1`，选择 2 | 新栈帧 `sub = {1,2}`，`start = 2` |
| 4 | `subsets(sub{1,2})` -> `subsets(sub{1,2,3})` | `{1,2}` 栈帧先把 `{1,2}` 加入答案，然后 `i=2`，选择 3 | 新栈帧 `sub = {1,2,3}`，`start = 3` |

这 4 步一直在“向下压栈”：每次执行到递归调用，就创建一个新的 `subsets()` 栈帧。到第 4 步时，调用栈大致是：

```text
main
└── subsets({})
    └── subsets({1})
        └── subsets({1,2})
            └── subsets({1,2,3})
```

### 第 2 组：从 `{1,2,3}` 返回，回到 `{1}` 后改走 `{1,3}`

| 画板步骤 | 从哪里到哪里 | 发生了什么 | 关键状态 |
| --- | --- | --- | --- |
| 5 | `subsets(sub{1,2,3})` -> `subsets(sub{1,2})` | `{1,2,3}` 栈帧进入后收集 `{1,2,3}`；因为 `start=3`，循环不执行，直接结束并返回 | `ans` 已有 `{1}`、`{1,2}`、`{1,2,3}` |
| 6 | `subsets(sub{1,2})` -> `subsets(sub{1})` | 回到 `{1,2}` 栈帧，递归调用结束，执行 `pop_back()` 撤销 3；这一层也没有更多数字可选，于是返回 `{1}` 栈帧 | `sub` 从 `{1,2,3}` 恢复为 `{1,2}`，随后回到 `{1}` |
| 7 | `subsets(sub{1})` -> `subsets(sub{1})` | 回到 `{1}` 栈帧后，先执行 `pop_back()` 撤销刚才选的 2 | `sub` 从 `{1,2}` 恢复为 `{1}` |
| 8 | `subsets(sub{1})` -> `subsets(sub{1})` | `{1}` 栈帧的 `for` 循环继续，`i` 从 1 增加到 2，准备尝试数字 3 | 仍在同一个 `{1}` 栈帧 |
| 9 | `subsets(sub{1})` -> `subsets(sub{1,3})` | 执行 `push 3`，递归进入 `{1,3}` 栈帧 | 新栈帧 `sub = {1,3}`，`start = 3` |
| 10 | `subsets(sub{1,3})` -> `subsets(sub{1})` | `{1,3}` 栈帧进入后收集 `{1,3}`；`start=3`，循环不执行，直接返回 | `ans` 增加 `{1,3}` |
| 11 | `subsets(sub{1})` -> `subsets(sub{1})` | 回到 `{1}` 栈帧，执行 `pop_back()` 撤销 3 | `sub` 从 `{1,3}` 恢复为 `{1}` |
| 12 | `subsets(sub{1})` -> `subsets(sub{1})` | `{1}` 栈帧的循环结束，准备退出这个函数 | 以 1 开头的分支全部完成 |
| 13 | `subsets(sub{1})` -> `subsets(sub{})` | `{1}` 栈帧结束，返回根栈帧 | 调用栈回到 `subsets({})` |

这一组是理解回溯的关键。递归返回后不是马上结束整个程序，而是回到上一层暂停的位置，先 `pop_back()` 恢复现场，再看同一层 `for` 循环还有没有下一个选择。

到第 13 步，所有以 1 开头的子集都已经走完：

```text
{1}, {1,2}, {1,2,3}, {1,3}
```

### 第 3 组：回到根栈帧，改选 2，再走到 `{2,3}`

| 画板步骤 | 从哪里到哪里 | 发生了什么 | 关键状态 |
| --- | --- | --- | --- |
| 14 | `subsets(sub{})` -> `subsets(sub{})` | 根栈帧里，递归 `{1}` 分支返回后，执行 `pop_back()` 撤销 1 | `sub` 从 `{1}` 恢复为 `{}` |
| 15 | `subsets(sub{})` -> `subsets(sub{})` | 根栈帧的 `for` 循环继续，`i` 从 0 增加到 1，准备尝试数字 2 | 仍在根栈帧 |
| 16 | `subsets(sub{})` -> `subsets(sub{2})` | 执行 `push 2`，递归进入 `{2}` 栈帧 | 新栈帧 `sub = {2}`，`start = 2` |
| 17 | `subsets(sub{2})` -> `subsets(sub{2,3})` | `{2}` 栈帧进入后收集 `{2}`；然后 `i=2`，选择 3，递归进入 `{2,3}` | 新栈帧 `sub = {2,3}`，`start = 3` |
| 18 | `subsets(sub{2,3})` -> `subsets(sub{2})` | `{2,3}` 栈帧进入后收集 `{2,3}`；`start=3`，没有后续数字，返回 `{2}` 栈帧 | `ans` 增加 `{2,3}` |
| 19 | `subsets(sub{2})` -> `subsets(sub{2})` | 回到 `{2}` 栈帧，执行 `pop_back()` 撤销 3 | `sub` 从 `{2,3}` 恢复为 `{2}` |
| 20 | `subsets(sub{2})` -> `subsets(sub{})` | `{2}` 栈帧循环结束，返回根栈帧 | 以 2 开头的分支完成 |

注意这里不会出现 `{2,1}`。原因是进入 `{2}` 栈帧时 `start=2`，后面只能尝试下标 2 的数字 3，不能回头再选 1。

### 第 4 组：根栈帧最后选 3，然后回到 `main()`

| 画板步骤 | 从哪里到哪里 | 发生了什么 | 关键状态 |
| --- | --- | --- | --- |
| 21 | `subsets(sub{})` -> `subsets(sub{})` | 根栈帧中，`{2}` 分支返回后，执行 `pop_back()` 撤销 2 | `sub` 从 `{2}` 恢复为 `{}` |
| 22 | `subsets(sub{})` -> `subsets(sub{})` | 根栈帧的 `for` 循环继续，`i` 从 1 增加到 2，准备尝试数字 3 | 仍在根栈帧 |
| 23 | `subsets(sub{})` -> `subsets(sub{3})` | 执行 `push 3`，递归进入 `{3}` 栈帧 | 新栈帧 `sub = {3}`，`start = 3` |
| 24 | `subsets(sub{3})` -> `subsets(sub{})` | `{3}` 栈帧进入后收集 `{3}`；`start=3`，没有后续数字，返回根栈帧 | `ans` 增加 `{3}` |
| 25 | `subsets(sub{})` -> `subsets(sub{})` | 根栈帧执行最后一次 `pop_back()`，撤销 3；根栈帧的循环也结束 | `sub` 恢复为 `{}` |
| 26 | `subsets(sub{})` -> `main()` | 根栈帧结束，返回 `main()`，整个递归过程完成 | call stack 清空，只剩 `main()` 收尾打印答案 |

到第 26 步，画板里的全部箭头都走完了。最终 `ans` 的顺序是：

```text
{}, {1}, {1,2}, {1,2,3}, {1,3}, {2}, {2,3}, {3}
```

也就是代码打印时看到的 8 个子集。

## 用树来看这 26 步

如果把每一次“进入函数”看成一个节点，递归树大概是这样：

```text
[]
├── [1]
│   ├── [1,2]
│   │   └── [1,2,3]
│   └── [1,3]
├── [2]
│   └── [2,3]
└── [3]
```

这棵树上的每个节点，都会在进入函数时被加入 `ans`。所以最终答案就是：

```text
[]
[1]
[1,2]
[1,2,3]
[1,3]
[2]
[2,3]
[3]
```

这和代码实际打印顺序一致。

## call stack 到底在做什么

递归的 call stack 可以理解成“还没做完的函数现场”。

当代码执行到：

```cpp
subsets(nums, sub, ans, i + 1);
```

当前函数会暂停，新的函数压入调用栈。等新函数跑完返回后，程序会回到刚才暂停的位置，继续执行下一句：

```cpp
sub.pop_back();
```

这就是为什么每次递归回来都要 `pop_back()`：它不是删除答案，而是把当前路径恢复到进入下一次选择之前的状态。

举个例子：

```text
sub = [1,2]
选择 3 -> sub = [1,2,3]
进入递归，收集 [1,2,3]
递归返回
pop 3 -> sub = [1,2]
```

如果不 `pop`，后面的分支就会带着不该带的数字继续走，结果会乱掉。

## 这段代码的核心模板

可以把它背成一个回溯模板：

```text
进入节点：记录当前路径
从 start 到末尾枚举选择：
    做选择：push
    递归进入下一层：start = i + 1
    撤销选择：pop
```

对应到代码就是：

```cpp
ans.emplace_back(sub);

for (int i = start; i < nums.size(); i++) {
    sub.emplace_back(nums[i]);
    subsets(nums, sub, ans, i + 1);
    sub.pop_back();
}
```

只要抓住这 4 行，就能看懂整个 26 步过程：

1. `ans.emplace_back(sub)`：当前路径就是一个子集。
2. `for (int i = start; ...)`：只往后选，避免重复。
3. `sub.emplace_back(nums[i])`：选择当前数字。
4. `sub.pop_back()`：递归回来后撤销选择，换下一条路。

## 最后总结

这段递归不是在一次性“算出”所有子集，而是在一棵选择树上深度优先遍历。

每一次进入函数，代表走到了一个新的节点；每一个节点上的 `sub`，都是一个合法子集；每一次返回后的 `pop_back()`，都是为了恢复现场，让同一层可以尝试下一个选择。

所以，递归求子集的本质就是：

```text
一路向下做选择，走到哪里就收集哪里的路径；
递归返回后撤销选择，再换下一条路。
```
