> 基于 `/media/wangpan/hdd1/0-me/zhihu` 项目的 `config.yaml`、`src/content_generator.py`（SYSTEM_PROMPT、主题池、book_reflection 选书方向）反推。

## 一、定位一句话

**给在城市里「看起来过得不错、但内心常常很累」的成年人，一个不说教、不打鸡血、像深夜可以呼吸一下的角落。**

他们要的不是答案，是被理解一下、被允许慢一点、被提醒「你不是只有你一个人这样」。

代码里写得最直白的一句（`SYSTEM_PROMPT`）：

> 读者是**在繁忙嘈杂社会中寻找心灵栖息地的成年人**。

## 二、目标用户画像

| 维度 | 描述 |
|---|---|
| **年龄** | 25–40 岁，知乎核心用户层 |
| **地域** | 一二线城市为主 |
| **身份** | 脑力劳动者：程序员 / 产品 / 设计 / 运营 / 投行 / 咨询 / 体制内中青年；研究生；文艺青年 |
| **生活状态** | 处在「上有 KPI 下有房贷」的夹层期，工作生活节奏快、信息密度高、社交账号多、真实倾诉对象少 |
| **阅读偏好** | 心理学、哲学、文学；爱看《被讨厌的勇气》《当下的力量》《也许你该找个人聊聊》《纳瓦尔宝典》这类书；不爱大字报式鸡汤，但又确实需要慰藉 |
| **行为习惯** | 深夜刷知乎 / 小红书；通勤路上戴耳机；习惯把情绪写在豆瓣 / 朋友圈「仅自己可见」 |
| **内容敏感点** | 一眼看出是水文 / 营销号 / AI 味的东西会立刻划走；讨厌鸡汤、口号、emoji 堆砌、对仗排比 |

旁证：`book_reflection` 模式选的书是心理学 / 哲学类，章节字数门槛 ≥ 800 字才算正文，明显是给愿意安静读字的人看的。

## 三、用户痛点（与主题池一一对应）

主题池里 15 条 themes 实际上是 15 类痛点的「症状名」，每一条背后都对应一群典型的内心处境：

| 痛点类别 | 典型内心独白 | 对应主题 |
|---|---|---|
| **信息过载、长期紧绷** | 「脑子一直停不下来，刷手机也没法放松」 | 在嘈杂世界里找到内心的安静；慢下来，才能听见自己 |
| **关系密集但深度孤独** | 「微信几千个好友，能说真话的没几个」 | 孤独不是问题，独处是能力 |
| **自我怀疑与完美主义** | 「我是不是哪里不够好；别人都比我跑得快」 | 接纳自己的不完美；成长不是变成别人，而是更像自己 |
| **慢性焦虑** | 「对未来发慌，但又使不上劲」 | 把焦虑转化为行动力；在不确定中安住 |
| **情绪不被允许** | 「脆弱是缺点，负面情绪要藏起来」 | 如何与负面情绪共处；允许自己脆弱 |
| **意义感缺失** | 「按成功剧本在跑，但越来越问『这是我要的吗』」 | 重新定义成功与幸福；向内探索：看见真实的自己 |
| **行动瘫痪** | 「道理都懂，就是动不了」 | 微小的坚持改变命运;把每一天活成第一天 |
| **被动受害者心态** | 「都是环境 / 原生家庭 / 老板让我这样的」 | 认知升级：从受害者到创造者 |
| **生活与物欲堆积** | 「东西、关系、念头全嫌多，想清空」 | 断舍离：清理生活也清理心灵 |

## 四、内容策略（反推自 SYSTEM_PROMPT）

为了对得上这群读者的敏感点，写作端定了几条硬约束：

**要做到的：**
- 真诚不矫情，温柔不软弱，理性中带温度
- 像深夜对话，不说教，不口号
- 用具体画面、感受、场景；段落短，留白多
- 偶尔用第二人称「你」拉近距离，不滥用
- 结尾留一个微小提醒或开放问题，不强行升华

**绝对不做的：**
- 鸡汤句式（「其实」「要相信」「加油」开头的口号）
- AI 味对仗排比、总结陈词
- 引用名人名言
- emoji、表情符号
- （book_reflection 模式）任何「读书笔记」式元叙述：「作者」「这一节」「书里」「读到这里」等

这些约束的本质，是替读者预先挡掉所有让他们「立刻识破并划走」的信号。

## 五、给运营的延伸思考

1. **内容选题**可以围绕这九类痛点轮换，避免同一类连发；目前 themes 是随机抽取，可考虑加去重 / 配额机制。
2. **发布时段**早 7:00 通勤前、晚 18:00 下班路上——契合目标用户「在缝隙里找呼吸」的使用场景。
3. **配图风格**已经定为 `ethereal, peaceful, minimalist, soft natural light, dreamy atmosphere, no text, no people`——抽象意境图，避开人脸 / 文字，规避「鸡汤海报感」。
4. **互动设计**（暂未实现）应延续「不打扰」基调：被点赞 / 被评论时的回复也要保持深夜对话感，不能突然变成营销号客服。
